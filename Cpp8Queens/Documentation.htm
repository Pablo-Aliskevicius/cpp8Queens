Fast implementation of the 8 queens problem in C++

Since ULONGLONG has 64 bits and the chess board has 64 cells, we can use 64-bit integers to speed up the 8-queens problem

We can select a 'row' by masking 8 bits of this unsigned long long. For instance, 0xff00 will mask the second row, and 0xff'0000 the third.

We can select a column, again, by using the proper mask. For instance, 0x2020'2020'2020'2020 will be the third column: the first (leftmost) will be 0x8080'8080'8080'8080.

The bits will be mapped to the board in the following order:

 7  6  5  4  3  2  1  0
15 14 13 12 11 10  9  8
23 22 21 20 19 18 17 16
31 30 29 28 27 26 25 24
39 38 37 36 35 34 33 32
47 46 45 44 43 42 41 40
55 54 53 52 51 50 49 48
63 62 61 60 59 58 57 56

The program will (for an 8 by 8 board):

    For each of the first 4 rows (the bottom 4 can be taken by using symmetry):
     
  1. Take the cell at [row, zero] (it is handling the first column here)
  2. 'Put a queen' there
  3. Threaten the relevant cells: that is, set the bits in the same row and diagonals to one.
  4. Handle the next column, recursively:
    1. If we are at the last column, we have a solution. Save the solution somewhere and break the recursion.
    2. If all the rows in the next column are threatened, we reached a dead end. Break the recursion.
    3. Get a list of non-threatened rows in the next column. That is, where the relevant bit is set to zero.
    4. For each of these rows:
      1. Threaten the corresponding cell at 'row, column'
      2. Recurse...
  • To 'threaten' a row, we just OR its bits with 0xff, properly shifted.
  • To 'threaten' a diagonal, we OR with the corresponding diagonal.
  • For fun, I used template metaprogramming to generate at compile time the 'threatening masks' for all 64 positions in the board.
  • To find out which rows are not threatened in a column, we AND it with the corresponding mask, and go over all 8 bytes - zeros are good.

It's also generalized for boards smaller than 8 by 8, down to 4x4 (smallest with a solution).
To generalize to bigger numbers (all the way up to 22x22), you need to use Intel Intrinsics and a CPU that supports AVX512. Mine doesn't.

Solutions

Each 'run' finds all the solutions for a given board size, and the first (size/2) columns, rounded up.

The runs are so fast that my old machine (2015, i5 with AVX2) doesn't always measure them, thus the zeros.
